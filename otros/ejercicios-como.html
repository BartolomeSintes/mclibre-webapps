Introducción

Cada una de las aplicaciones web que se tratan en este curso son aplicaciones grandes, con muchas opciones y posibilidades.

El objetivo de este curso no es trabajar en profundidad ninguna aplicación en concreto, sino practicar la instalación de diferentes aplicaciones y explorar y entender su forma de funcionamiento, con el objetivo de que el alumno adquiera la destreza de instalar y administrar otras aplicaciones por su cuenta.

Para cada aplicación se proponen algunas actividades básicas, aunque naturalmente se aconseja al alumno que profundice cuanto le sea posible en las aplicaciones que encuentre más interesantes.

Se recomienda al alumno que primero intente realizar las actividades por sí mismo. En muchos casos se necesita información adicional, que puede encontrarse buscando en Internet. Si no se consigue realizar alguna actividad o no se encuentra fácilmente una respuesta en Internet, se pueden consultar las páginas de soluciones que contienen en muchos casos indicaciones paso a paso, ilustradas con capturas.

Probablemente, en las primeras aplicaciones el alumno necesitará consultar las soluciones pero, a medida que vaya familiarizándose con las diferentes aplicaciones, le resultará más fácil resolver las actividades sin recurrir a las soluciones. El tiempo empleado explorando las posibilidades de las aplicaciones, buscando y probando la forma de hacer las cosas, estará siempre bien empleado.

Nota:

  • En el curso 2011/12, en que comencé a elaborar estos apuntes, los apuntes estaban organizados como tutoriales en los que ejercicios y soluciones estaban juntos.
  • En el curso 2012/13 separé ejercicios y soluciones en páginas distintas.
  • En el curso 2013/14 empecé a redactar los ejercicios de manera que el objetivo fuera crear sitios ficticios dedicados a las aplicaciones web.
  • En el curso 2014/15 continué el trabajo del año anterior. Dejé de dar dos aplicaciones (phpBB y MediaWiki) para poder ampliar el tiempo dedicado a WordPress y Drupal. Amplié la parte dedicada a la instalación de aplicaciones con Bitnami.
  • En el curso 2015/16 continué el trabajo del año pasado en la redacción de los ejercicios. Actualizar los apuntes a las nuevas versiones publicadas desde el curso pasado me ocupó la mayor parte del tiempo, especialmente las dedicadas a Drupal (debido a la publicación de Drupal 8).

Aplicaciones web y bases de datos

Todas las aplicaciones que se verán en este curso guardan la información en una base de datos. Todas pueden trabajar con MySQL (o MariaDB), por lo que trabajaremos siempre con la base de datos MySQL/MariaDB que instala XAMPP, como se comenta en la lección Configuración general.

Para poder acceder a la base de datos, las aplicaciones necesitan utilizar un usuario de la base de datos y conocer su contraseña. Por motivos de seguridad, el usuario de la base de datos empleado por las aplicaciones no debe ser el usuario root de la base de datos. En estos apuntes se propone que cada aplicación utilice un usuario distinto, aunque un usuario podría perfectamente administrar varias aplicaciones.

Para crear los usuarios de la base de datos que utilizará cada aplicación, utilizaremos phpMyAdmin, una aplicación web que permite administrar gráficamente MySQL/MariaDB. Para realizar esas tareas de administración generales, phpMyAdmin se conectará a la base de datos utilizando el usuario root de la base de datos, pero si resulta necesario utilizar phpMyAdmin para realizar las tareas de administración específicas de cada aplicación, entonces utilizaremos los usuarios específicos.

Para practicar el uso de phpMyAdmin, la primera aplicación que se trabaja en este curso es precisamente phpMyAdmin. Pero hay que tener en cuenta que:

  • Por un lado, XAMPP incluye una versión de phpMyAdmin, que podemos utilizar sin necesidad de instalarla y que está ya configurada para conectarse como root a la base de datos
  • Por otro lado, descargaremos e instalaremos una versión más reciente de phpMyAdmin, con la que podremos conectarnos como root o como cualquiera de los usuarios que vayamos creando.

Cuando trabajemos con el resto de aplicaciones, el proceso será siempre similar:

  • como usuario root de la base de datos, primero crearemos con phpMyAdmin el usuario de la aplicación.

    Este paso lo podríamos hacer tanto con el phpMyAdmin de XAMPP como el phpMyAdmin instalado por nosotros, pero en este curso usaremos preferentemente el nuestro.

  • como usuario de la aplicación, instalaremos la aplicación.
  • como usuario de la aplicación, utilizaremos en algunos casos phpMyAdmin para algunas tareas (como copias de seguridad, restauraciones, etc.).

    Este paso lo haremos con nuestro phpMyAdmin ya que utilizaremos los usuarios de cada aplicación, aunque también podría utilizarse el phpMyAdmin de XAMPP con usuario root.

Tipos de usuarios

Al trabajar con aplicaciones web, intervienen diferentes tipos de usuarios:

  • el administrador del servidor, que se encarga de que el servidor web y el sistema gestor de bases de datos funcionen correctamente.

    En estos apuntes, ese usuario es único y se le da el nombre de root

    Como usuario root como mínimo tendremos que crear los usuarios de las aplicaciones.

  • el usuario de la base de datos, con el que la aplicación se conecta a la base de datos.

    En estos apuntes, crearemos un usuario distinto para cada aplicación y el nombre dependerá de la aplicación: iaw_wordpress, iaw_opencart, etc. En caso de hacer varias instalaciones de una aplicación, se crearán distintos usuarios: iaw_opencart_1, iaw_opencart_2, etc.

    En algunas aplicaciones, como usuario iaw_aplicacion no tendremos que realizar ninguna tarea, ya que es la propia aplicación la que gestiona al usuario, pero en otras, lo utilizaremos para conectarnos con la base de datos y realizar tareas de administración de la base de datos (copias de seguridad, etc.).

  • el administrador de cada aplicación, que se encarga de personalizar y administrar cada aplicación.

    En estos apuntes, a los administradores se les da normalmente el nombre de admin.

    Como administradores de la aplicación instalaremos la aplicación y la administraremos mediante la propia aplicación.

  • los usuarios registrados de cada aplicación, que pueden crear contenido en cada aplicación. Normalmente, existen diferentes tipos de usuarios que pueden realizar más o menos tareas.

    En estos apuntes, estos usuarios se les da un nombre formado por la categoría a la que pertenecen y un número (editor_1, cliente_1, etc.). Estos nombres dependen de la aplicación de que se trate pues cada aplicación define unas categorías distintas.

    Como usuarios registrados de la aplicación podremos llevar a cabo las tareas que nos permita la categoría de usuario (crear contenido, modificar contenido, etc.).

Al realizar los ejercicios, tenemos que ir jugando cada uno de estos papeles. Algunas tareas pueden realizarse con más de un usuario, pero normalmente cada tarea la tiene que realizar un usuario en concreto. Tenemos que ser conscientes de qué usuario somos en cada momento e ir cambiando de usuario en caso necesario. Para no tener que estar cerrando sesiones e identificándonos continuamente, una solución es utilizar navegadores distintos para cada tarea. Por ejemplo, tener abierta en Firefox una sesión de administrador de la aplicación, tener abierta en Chrome una sesión de usuario registrado y usar Internet Explorer para navegar la aplicación como usuario no registrado.
