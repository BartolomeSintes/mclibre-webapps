Configuración para Windows

Servidores

En este curso se utilizará la misma instalación de Apache utilizada en el curso de Programación web en PHP. En el apartado Software > XAMPP de este curso se explica la instalación y configuración de Apache y MySQL (mediante XAMPP).

Para poder ejecutar las aplicaciones web, se deberá iniciar previamente el servidor Apache y el servidor MySQL desde el panel de control de XAMPP.

Algunas aplicaciones requieren modificaciones de la configuración de Apache o PHP y se comentan en las páginas correspondientes, pero hay un cambio que conviene hacer para no tener problemas cuando una página tarda mucho tiempo en ejecutarse (lo que puede ocurrir durante la instalación de las aplicaciones o con tareas como actualizaciones o copias de seguridad).

Directiva short_open_tag

La directiva short_open_tag indica la tiempo máximo de ejecución de una página PHP. Si la página tarda más de ese tiempo en ejecutarse, el servidor interrumpirá la ejecución del programa. En XAMPP 7.1 esta directiva viene predeterminada con el valor 30 (segundos). Para evitar ese problema, se recomienda aumentar ese valor, por ejemplo a una hora

max_execution_time=3600        ; Valor recomendado en este curso

Carpetas

Las aplicaciones web se descargarán como archivos comprimidos que se deberán descomprimir en la misma carpeta, por ejemplo, en:

  • Mis documentos > IAW Nombre-de-alumno > Aplicaciones

Cada aplicación se guardará en su propia carpeta. Se recomienda que el nombre de la carpeta sea simplemente el nombre de la aplicación, en minúsculas y sin incluir el número de versión. Únicamente cuando se instalen varias versiones de la misma aplicación, cada una en supropia carpeta, se añadirá un número al nombre de la aplicación (_1, _2, etc.)

Por ejemplo, en el caso de la aplicación web phpMyAdmin se creará la carpeta:

  • Mis documentos > IAW Nombre-de-alumno > Aplicaciones > phpmyadmin

Para poder ejecutar las aplicaciones, se deberá crear un alias en el archivo de configuración de Apache httpd.conf. El alias puede ser simplemente /webapps y apuntar a la carpeta Aplicaciones, de manera que un único alias sea suficiente para todas las aplicaciones.

Edición del código fuente de las aplicaciones

En general, no se editará el código fuente de las aplicaciones web, pero ocasionalmente si que se modificará algún fichero en concreto.

Para esas modificaciones puntuales, se recomienda utilizar un editor ligero como Notepad++.

También se puede utilizar el editor Eclipse for PHP developers, aunque como Eclipse PHP revisa y comprueba todos los archivos, a medida que vayamos instalando aplicaciones web, ese proceso tardará cada vez más tiempo.

Si se quiere utilizar Eclipse PHP, se recomienda crear un workspace específico. Se recomienda:

  • crear el workspace en "C:\software\workspace-nombre-de-alumno-php"
  • crear un proyecto que se llame: Nombre-de-alumno (sin espacios en el nombre del alumno)
  • crear una carpeta enlazada que se llame webapps y enlace a la carpeta Mis documentos > IAW Nombre-de-alumno > Aplicaciones

Al descomprimir una nueva aplicación web se deberá actualizar la vista del Explorador PHP de Eclipse.

Para poder ejecutar las aplicaciones desde Eclipse PHP, se deberá crear otro alias en el archivo de configuración de Apache httpd.conf. La ruta del alias será /Nombre-de-alumno/webapps, de manera que un único alias sea suficiente para todas las aplicaciones.

Al ejecutar la página principal de cada aplicación web desde Eclipse PHP, se abrirá la página web en el navegador. Por ejemplo, en el caso de phpMyAdmin,

  • http://localhost/Nombre-de-alumno/webapps/phpmyadmin/

Página web índice de aplicaciones

Cree una página web index.html en la carpeta Aplicaciones que sirva de índice para entrar en las diferentes aplicaciones sin necesidad de entrar en Eclipse. Por ejemplo:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="utf-8" />
  <title>Aplicaciones web</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
</head>

<body>
  <h1>Aplicaciones web</h1>

  <h2>Enlaces a aplicaciones</h2>

  <ul>
    <li><a href="http://localhost/webapps/phpmyadmin/">phpMyAdmin</a></li>
  </ul>

  <h2>Alias a aplicaciones</h2>

  <ul>
    <li><a href="http://localhost/phpmyadmin2/">phpMyAdmin</a></li>
  </ul>
</body>
</html>
