Qué es Ansible

Ansible es una herramienta de administración automatizada de ordenadores, creada por Michael DeHaan en 2012. La primera versión se publicó en marzo de 2012 y la versión 1.0 se publicó en febrero de 2013. RedHat compró Ansible en octubre de 2015.

Con Ansible, un ordenador es la máquina de control (control machine) y el resto de ordenadores son los nodos administrados (managed nodes).

En la máquina de control es donde se instala Ansible. En los nodos administrados no es necesario instalar ingún software. El acceso de la máquina de control a los nodos administrados se hace normalmente mediante conexiones SSH.

Ejemplos

Máquina de control: instalación de Ansible en Ubuntu 18.04

Instalaremos Ansible en una máquina virtual de VirtualBox con Ubuntu 18.04 Server que ejercerá de máquina de control. La instalación la realizaremos con Packer.

Para instalar Ansible con Packer, necesitamos:

  • Haber creado una imagen de la máquina virtual e instalado Ubuntu Server en ella (véase el ejemplo 1 de la lección de Packer)
  • Crear la plantilla de packer (por ejemplo, ansible-1-template.json) y el script de shell a ejecutar (por ejemplo, ansible-1-shell.sh).
  • Ejecutar packer 
    C:\packer>packer build ansible-1-template.json

Si el proceso termina correctamente, Packer creará el directorio output-virtualbox-ovf y creará en él el fichero ovf (u ova) con una nueva imagen de máquina virtual.

Plantilla de creación de la máquina de control (ansible-1-plantilla.json)

Esta podría ser la plantilla de creación de la máquina de control Ubuntu:

{
  "builders": [
    {
      "type": "virtualbox-ovf",
      "source_path": "output-virtualbox-iso/packer-virtualbox-iso-1520842303.ovf",
      "boot_wait": "30s",
      "ssh_username": "barto",
      "ssh_password": "barto",
      "shutdown_command": "echo 'barto' | sudo -S shutdown -P now",
      "post_shutdown_delay": "120s",
      "format": "ova",
      "vboxmanage_post": [
        ["modifyvm", "{{.Name}}", "--nic1", "bridged"],
        ["modifyvm", "{{.Name}}", "--bridgeadapter1", "Realtek PCIe GBE Family Controller"],
        ["modifyvm", "{{.Name}}", "--vram", "64"]
      ]
    }
  ],
  "provisioners": [
    {
      "type": "shell",
      "execute_command": "echo 'barto' | sudo -S sh -c '{{ .Vars }} {{ .Path }}'",
      "script": "ansible-1-shell.sh",
      "pause_before": "10s"
    }
  ]
}

Notas:

  • El retardo tras el apagado (línea 10) es para dar tiempo a VirtualBox a desbloquear la máquina y poder hacer la exportación final.
  • El nombre del adaptador 1 (línea 14) depende del ordenador en el que esté instalado VirtualBox.

Este podría ser el script de instalación de Ansible [fuente].

sudo apt-get update
sudo apt-get upgrade -y
sudo apt-get install software-properties-common
sudo apt-add-repository -y ppa:ansible/ansible
sudo apt-get update
sudo apt-get install -y ansible
